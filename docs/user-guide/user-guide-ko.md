## 1. 제품 소개

**Samba 4 AD Manager**는 Samba 4 Active Directory 도메인 컨트롤러를 웹에서 관리하는 포털입니다. 사용자, 그룹, 컴퓨터, 조직 단위, 그룹 정책, DNS 영역 및 도메인 보안 정책을 관리할 수 있는 직관적인 인터페이스를 제공하여 Windows Server 관리 도구의 필요성을 대체합니다.

### 주요 기능

- **통합 AD 관리**: 사용자, 그룹, 컴퓨터, OU, GPO
- **DNS 관리**: 정방향/역방향 영역 관리
- **보안 정책**: 비밀번호 및 계정 잠금 정책 설정
- **실시간 모니터링**: 시스템 상태, 서비스 상태, 보안 알림을 보여주는 대시보드
- **REST API**: JWT 인증 기반의 프로그래밍 방식 접근
- **다국어 UI**: 한국어 인터페이스 (API 응답은 영어)
- **오픈 소스**: Apache-2.0 라이선스

## 3. 설치 방법

### 빠른 설치

```bash
# 저장소 복제
git clone https://github.com/lhjnano/samba4-ad.git
cd samba4-ad

# 설치 스크립트 실행
chmod +x install.sh
sudo ./install.sh
```

설치 스크립트는 다음 작업을 수행합니다:
1. Samba 4 및 관련 패키지 설치 (`apt`)
2. Python 가상환경 생성
3. 모든 Python 의존성 패키지 설치
4. React 프론트엔드 빌드
5. systemd 서비스 구성 및 활성화
6. 포트 8000에서 웹 포털 시작

### 수동 설치

```bash
# 시스템 의존성 설치
sudo apt update
sudo apt install -y samba smbclient winbind ldap-utils python3-venv nodejs npm

# 백엔드 설정
cd backend
python3 -m venv .venv
.venv/bin/pip install -e .

# 프론트엔드 빌드
cd ../frontend
npm install
npm run build

# 서버 시작
cd ../backend
../backend/.venv/bin/uvicorn src.main:app --host 0.0.0.0 --port 8000
```

### systemd 서비스 관리

```bash
sudo systemctl start samba-ad-manager     # 시작
sudo systemctl stop samba-ad-manager      # 중지
sudo systemctl restart samba-ad-manager   # 재시작
sudo systemctl status samba-ad-manager    # 상태 확인
sudo systemctl enable samba-ad-manager    # 부팅 시 자동 시작
```

## 4. 초기 설정 (도메인 구축)

포털에 처음 접속하면 도메인이 구축되지 않은 경우 **설정 마법사**가 나타납니다.

### 설정 마법사 단계

1. **도메인 정보 입력**
   - **도메인 이름**: AD 도메인 (예: `corp.example.com`)
   - **NetBIOS 이름**: 짧은 도메인 이름 (예: `CORP`, 최대 15자)
   - **관리자 비밀번호**: 초기 도메인 관리자 비밀번호 (최소 8자)

2. **DNS 설정**
   - **DNS 전달자**: 상위 DNS 서버 (예: `8.8.8.8`)
   - DC 자체가 도메인의 주 DNS가 됩니다

3. **검토 및 구축**
   - 모든 설정 확인
   - **도메인 구축** 버튼 클릭

4. **완료**
   - `samba-tool domain provision` 실행
   - Samba 서비스 재시작
   - 로그인 페이지로 이동

> **참고**: 구축에는 1~3분이 소요됩니다. 진행 중에 브라우저를 닫지 마세요.

| 항목 | 값 |
|------|------|

## 6. 대시보드

대시보드는 Active Directory 환경의 전체 현황을 한눈에 보여줍니다.

### 구성

#### 통계 카드 (상단)
- **총 사용자**: 도메인 내 모든 사용자 계정
- **그룹**: 보안 그룹 및 배포 그룹
- **컴퓨터**: 도메인에 가입된 머신
- **조직 단위**: 컨테이너 구조
- **GPO**: 그룹 정책 개체
- **도메인 컨트롤러**: DC 수

#### 로그인 추이 차트
- 최근 7일간의 로그인 활동 막대 차트
- 일별 인증 이벤트 수 표시

#### 시스템 리소스
- **CPU 사용률**: 현재 프로세서 사용량
- **메모리**: RAM 사용률
- **디스크**: 저장공간 사용량

#### 서비스 상태
주요 AD 서비스의 실시간 상태:
- LDAP (포트 389)
- Kerberos (포트 88)
- DNS (포트 53)
- SMB (포트 445)

#### OU별 사용자 분포
조직 단위별 사용자 수 — 시각적 막대 차트.

#### 최근 알림
최신 보안 이벤트 (계정 잠금, 로그인 실패 등).

## 7. 사용자 관리

접근: **사이드바 → 사용자**

### 사용자 목록
- 페이지당 50명 표시
- 항목: 사용자 이름, 표시 이름, 이메일, 부서, 상태, OU, 최근 로그인
- 행 클릭 시 사용자 상세 정보 확인

### 검색 및 필터
- **검색 바**: 사용자 이름, 표시 이름 또는 이메일로 필터링
- **OU 필터**: 특정 OU의 사용자 표시
- **상태 필터**: 전체 / 활성 / 비활성 / 잠김

### 작업

#### 사용자 추가
1. **+ 사용자 추가** 클릭
2. 정보 입력:
   - **사용자 이름** (필수): 로그인 ID (예: `hkim`)
   - **표시 이름** (필수): 전체 이름 (예: `김현수`)
   - **이메일**: 이메일 주소
   - **부서**: 부서명
   - **직책**: 직위
   - **OU**: 대상 조직 단위
   - **비밀번호**: 초기 비밀번호 (최소 8자)
3. **생성** 클릭

#### 사용자 편집
1. 사용자 행 클릭
2. 드로어에서 필드 수정
3. **저장** 클릭

#### 비밀번호 재설정
1. 사용자 선택
2. **비밀번호 재설정** 클릭
3. 새 비밀번호 입력
4. 확인

#### 비활성화 / 활성화
1. 사용자 선택
2. **비활성화** 또는 **활성화** 클릭

#### 사용자 삭제
1. 사용자 선택
2. **삭제** 클릭
3. 확인

> **주의**: 삭제는 되돌릴 수 없습니다. 나중에 필요할 수 있다면 계정을 비활성화하세요.

#### CSV 내보내기
**내보내기** 클릭하여 모든 사용자를 CSV 파일로 다운로드.

## 12. DNS 관리

접근: **사이드바 → DNS**

### 영역 목록
- 정방향 영역 (예: `corp.example.com`)
- 역방향 영역 (예: `1.168.192.in-addr.arpa`)

### 작업

#### 레코드 보기
1. 영역 선택
2. 모든 DNS 레코드 확인 (A, CNAME, MX, SRV, TXT, PTR, SOA, NS)

#### 레코드 추가
1. 영역 선택
2. **+ 레코드 추가** 클릭
3. 정보 입력:
   - **이름**: 레코드 이름 (영역 루트는 `@` 사용)
   - **유형**: 레코드 유형 (A, CNAME, MX 등)
   - **값**: IP 주소 또는 호스트명
   - **TTL**: Time to Live (기본값: 3600)
4. **생성** 클릭

#### 레코드 삭제
1. 레코드 찾기
2. 휴지통 아이콘 클릭
3. 삭제 확인

## 16. API 참조

전체 대화형 API 문서는 다음에서 확인할 수 있습니다:
- **Swagger UI**: `http://<서버IP>:8000/docs`
- **ReDoc**: `http://<서버IP>:8000/redoc`
- **OpenAPI JSON**: `http://<서버IP>:8000/openapi.json`

### 인증

`/health`, `/auth/login`, `/setup/*`를 제외한 모든 API 엔드포인트에는 JWT Bearer 토큰이 필요합니다:

```bash
# 토큰 받기
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin"}'

# 토큰 사용
curl http://localhost:8000/api/v1/users \
  -H "Authorization: Bearer <토큰>"
```

## 17. 문제 해결

### 서비스가 시작되지 않는 경우

```bash
# 서비스 상태 확인
sudo systemctl status samba-ad-manager

# 로그 확인
sudo journalctl -u samba-ad-manager -n 50

# 포트 8000 사용 여부 확인
sudo ss -tlnp | grep 8000
```

### 웹 포털에 접속할 수 없는 경우

1. 서비스 실행 확인: `systemctl is-active samba-ad-manager`
2. 방화벽 규칙 확인: `sudo ufw status`
3. 네트워크 연결 확인: `ping <서버IP>`
4. `.env` 파일 설정 확인

### 로그인 실패

- **Mock 모드**: 기본 계정은 `admin`/`admin`
- **LDAP 모드**: Samba DC 실행 확인: `samba -V`
- LDAP 서비스 연결 확인: `ldapsearch -x -H ldap://localhost`

### 프론트엔드가 빈 페이지로 표시되는 경우

1. 프론트엔드 재빌드: `cd frontend && npm run build`
2. 서비스 재시작: `sudo systemctl restart samba-ad-manager`
3. 브라우저 캐시 삭제

### Mock 모드로 되돌리기

`.env` 파일 수정:
```
APP_MODE=mock
```
재시작: `sudo systemctl restart samba-ad-manager`
