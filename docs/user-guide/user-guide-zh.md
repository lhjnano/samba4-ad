## 1. 产品介绍

**Samba 4 AD Manager**是一个基于Web的管理门户，用于管理Samba 4 Active Directory域控制器。它提供直观的界面来管理用户、组、计算机、组织单位、组策略、DNS区域和域级安全策略，从而替代对Windows Server管理工具的需求。

### 主要功能

- **统一AD管理**：用户、组、计算机、OU、GPO
- **DNS管理**：正向和反向区域管理
- **安全策略**：密码和锁定策略配置
- **实时监控**：包含系统健康、服务状态和警报的仪表板
- **REST API**：基于JWT认证的编程式访问
- **多语言UI**：韩语界面（API响应为英语）
- **开源**：Apache-2.0许可证

## 3. 安装方法

### 快速安装

```bash
# 克隆仓库
git clone https://github.com/lhjnano/samba4-ad.git
cd samba4-ad

# 运行安装程序
chmod +x install.sh
sudo ./install.sh
```

安装程序将执行以下操作：
1. 通过`apt`安装Samba 4及相关软件包
2. 创建Python虚拟环境
3. 安装所有Python依赖
4. 构建React前端
5. 配置并启用systemd服务
6. 在端口8000上启动Web门户

### systemd服务管理

```bash
sudo systemctl start samba-ad-manager     # 启动
sudo systemctl stop samba-ad-manager      # 停止
sudo systemctl restart samba-ad-manager   # 重启
sudo systemctl status samba-ad-manager    # 查看状态
sudo systemctl enable samba-ad-manager    # 开机自启
```

## 4. 初始设置（域配置）

首次访问门户时，如果尚未配置域，将显示**设置向导**。

### 设置向导步骤

1. **域信息**
   - **域名**：您的AD域（例如：`corp.example.com`）
   - **NetBIOS名称**：短域名（例如：`CORP`，最多15个字符）
   - **管理员密码**：初始域管理员密码（最少8个字符）

2. **DNS配置**
   - **DNS转发器**：上游DNS服务器（例如：`8.8.8.8`）
   - DC本身将成为域的主DNS

3. **审查并配置**
   - 确认所有设置
   - 点击**配置域**开始

4. **完成**
   - 系统运行`samba-tool domain provision`
   - 重启Samba服务
   - 重定向到登录页面

> **注意**：配置需要1-3分钟。在此过程中请勿关闭浏览器。

## 5. 登录

### 默认凭据（Mock模式）

| 字段 | 值 |
|------|------|
| 用户名 | `admin` |
| 密码 | `admin` |

### 登录流程

1. 导航到`http://<服务器IP>:8000`
2. 输入用户名和密码
3. 点击**登录**
4. 将被重定向到仪表板

### 会话管理

- JWT令牌在**8小时**后过期
- 令牌过期时自动重定向到登录页
- 点击右上角的**退出**按钮手动结束会话

## 6. 仪表板

仪表板提供Active Directory环境的概览。

### 组成部分

#### 统计卡片（顶部）
- **用户总数**：域中所有用户账户
- **组**：安全组和通讯组
- **计算机**：已加入域的机器
- **组织单位**：容器结构
- **GPO**：组策略对象
- **域控制器**：DC数量

#### 登录趋势图
- 7天登录活动柱状图
- 显示每日身份验证事件总数

#### 系统资源
- **CPU使用率**：当前处理器利用率
- **内存**：RAM使用百分比
- **磁盘**：存储使用量

#### 服务状态
关键AD服务的实时健康状况：
- LDAP（端口389）
- Kerberos（端口88）
- DNS（端口53）
- SMB（端口445）

#### OU用户分布
每个组织单位的用户数 — 可视化柱状图。

#### 最近警报
最新安全事件（账户锁定、登录失败等）。

## 7. 用户管理

访问：**侧边栏 → 用户**

### 用户列表
- 分页列表（默认每页50条）
- 列：用户名、显示名称、邮箱、部门、状态、OU、最后登录

### 搜索与筛选
- **搜索栏**：按用户名、显示名称或邮箱筛选
- **OU筛选**：显示特定OU中的用户
- **状态筛选**：全部 / 活跃 / 禁用 / 锁定

### 操作

#### 创建用户
1. 点击**+ 添加用户**
2. 填写表单：
   - **用户名**（必填）：登录ID（例如：`wzhang`）
   - **显示名称**（必填）：全名（例如：`张伟`）
   - **邮箱**：电子邮件地址
   - **部门**：部门名称
   - **职位**：职务
   - **OU**：目标组织单位
   - **密码**：初始密码（最少8个字符）
3. 点击**创建**

#### 编辑用户
1. 点击用户行
2. 在抽屉中修改字段
3. 点击**保存**

#### 重置密码
1. 选择用户
2. 点击**重置密码**
3. 输入新密码

#### 禁用/启用用户
1. 选择用户
2. 点击**禁用**或**启用**

#### 删除用户
1. 选择用户
2. 点击**删除**
3. 确认操作

> **警告**：删除是永久的。如果以后可能需要该账户，请改为禁用。

#### 导出CSV
点击**导出**将所有用户下载为CSV文件。

## 8. 组管理

访问：**侧边栏 → 组**

### 组列表
- 列：名称、描述、类型、范围、成员数、OU

### 操作

#### 创建组
1. 点击**+ 添加组**
2. 输入信息：
   - **名称**（必填）：组名称
   - **描述**：组用途
   - **类型**：安全组或通讯组
   - **范围**：域本地、全局或通用
   - **OU**：目标OU
3. 点击**创建**

#### 管理成员
1. 点击组
2. 导航到**成员**标签页
3. 添加或删除用户DN

#### 导出CSV
下载所有组为CSV文件。

## 12. DNS管理

访问：**侧边栏 → DNS**

### 区域列表
- 正向区域（例如：`corp.example.com`）
- 反向区域（例如：`1.168.192.in-addr.arpa`）

### 操作

#### 查看记录
1. 选择区域
2. 查看所有DNS记录（A、CNAME、MX、SRV、TXT、PTR、SOA、NS）

#### 添加记录
1. 选择区域
2. 点击**+ 添加记录**
3. 输入信息：
   - **名称**：记录名称（区域根目录使用`@`）
   - **类型**：记录类型（A、CNAME、MX等）
   - **值**：IP地址或主机名
   - **TTL**：生存时间（默认：3600）
4. 点击**创建**

#### 删除记录
1. 找到记录
2. 点击垃圾桶图标
3. 确认删除

## 13. 域策略

访问：**侧边栏 → 策略**

### 密码策略

| 设置 | 描述 | 默认值 |
|------|------|--------|
| 复杂密码 | 要求大小写、数字、符号 | 启用 |
| 最小长度 | 最小密码字符数 | 7 |
| 密码历史 | 记住最近N个密码 | 24 |
| 最大使用期限（天） | 强制密码更改间隔 | 42 |
| 最小使用期限（天） | 密码可更改前的天数 | 1 |

### 账户锁定策略

| 设置 | 描述 | 默认值 |
|------|------|--------|
| 锁定阈值 | 锁定前的失败尝试次数 | 0（禁用） |
| 锁定持续时间（分钟） | 账户保持锁定的时间 | 30 |
| 重置计数器（分钟） | 计数器重置时间 | 30 |

### 更新策略
1. 修改所需字段
2. 点击**保存更改**
3. 更改对新会话立即生效

## 16. API参考

完整的交互式API文档可在以下地址查看：
- **Swagger UI**：`http://<服务器IP>:8000/docs`
- **ReDoc**：`http://<服务器IP>:8000/redoc`
- **OpenAPI JSON**：`http://<服务器IP>:8000/openapi.json`

### 认证

除`/health`、`/auth/login`和`/setup/*`外，所有API端点都需要JWT Bearer令牌：

```bash
# 获取令牌
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin"}'

# 使用令牌
curl http://localhost:8000/api/v1/users \
  -H "Authorization: Bearer <您的令牌>"
```

### 主要端点

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/api/v1/auth/login` | 登录 |
| GET | `/api/v1/dashboard/stats` | 仪表板概要 |
| GET | `/api/v1/users` | 用户列表 |
| POST | `/api/v1/users` | 创建用户 |
| GET | `/api/v1/groups` | 组列表 |
| GET | `/api/v1/computers` | 计算机列表 |
| GET | `/api/v1/ou` | OU列表 |
| GET | `/api/v1/gpo` | GPO列表 |
| GET | `/api/v1/dns/zones` | DNS区域列表 |
| GET | `/api/v1/logs` | 系统日志 |
| GET | `/api/v1/policies/domain` | 域策略 |

## 17. 故障排除

### 服务无法启动

```bash
# 检查服务状态
sudo systemctl status samba-ad-manager

# 查看日志
sudo journalctl -u samba-ad-manager -n 50

# 检查端口8000是否被占用
sudo ss -tlnp | grep 8000
```

### 无法访问Web门户

1. 验证服务正在运行：`systemctl is-active samba-ad-manager`
2. 检查防火墙规则：`sudo ufw status`
3. 验证网络连接：`ping <服务器IP>`
4. 检查`.env`文件设置

### 登录失败

- **Mock模式**：默认凭据为`admin`/`admin`
- **LDAP模式**：验证Samba DC正在运行：`samba -V`

### 前端显示空白页

1. 重新构建前端：`cd frontend && npm run build`
2. 重启服务：`sudo systemctl restart samba-ad-manager`
3. 清除浏览器缓存

### 恢复到Mock模式

编辑`.env`文件：
```
APP_MODE=mock
```
重启：`sudo systemctl restart samba-ad-manager`
