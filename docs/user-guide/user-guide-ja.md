## 1. 製品紹介

**Samba 4 AD Manager**は、Samba 4 Active DirectoryドメインコントローラーをWebブラウザから管理するポータルです。ユーザー、グループ、コンピューター、組織単位、グループポリシー、DNSゾーン、ドメインセキュリティポリシーを管理する直感的なインターフェースを提供し、Windows Server管理ツールの必要性を置き換えます。

### 主な機能

- **統合AD管理**: ユーザー、グループ、コンピューター、OU、GPO
- **DNS管理**: フォワードゾーン・リバースゾーン管理
- **セキュリティポリシー**: パスワード・アカウントロックアウトポリシー設定
- **リアルタイム監視**: システム状態、サービス状態、セキュリティアラートを表示するダッシュボード
- **REST API**: JWT認証によるプログラマティックアクセス
- **多言語UI**: 韓国語インターフェース（API応答は英語）
- **オープンソース**: Apache-2.0ライセンス

## 3. インストール方法

### クイックインストール

```bash
# リポジトリのクローン
git clone https://github.com/lhjnano/samba4-ad.git
cd samba4-ad

# インストーラーの実行
chmod +x install.sh
sudo ./install.sh
```

インストーラーは以下の作業を行います:
1. Samba 4および関連パッケージのインストール（`apt`）
2. Python仮想環境の作成
3. 全Python依存パッケージのインストール
4. Reactフロントエンドのビルド
5. systemdサービスの設定と有効化
6. ポート8000でのWebポータル起動

### systemdサービス管理

```bash
sudo systemctl start samba-ad-manager     # 開始
sudo systemctl stop samba-ad-manager      # 停止
sudo systemctl restart samba-ad-manager   # 再起動
sudo systemctl status samba-ad-manager    # 状態確認
sudo systemctl enable samba-ad-manager    # 自動起動設定
```

## 4. 初期設定（ドメイン構築）

ポータルに初回アクセス時、ドメインが構築されていない場合、**セットアップウィザード**が表示されます。

### セットアップウィザード手順

1. **ドメイン情報入力**
   - **ドメイン名**: ADドメイン（例: `corp.example.com`）
   - **NetBIOS名**: 短いドメイン名（例: `CORP`、最大15文字）
   - **管理者パスワード**: 初期ドメイン管理者パスワード（最小8文字）

2. **DNS設定**
   - **DNSフォワーダー**: 上位DNSサーバー（例: `8.8.8.8`）

3. **確認と構築**
   - 全設定を確認
   - **ドメイン構築**ボタンをクリック

4. **完了**
   - `samba-tool domain provision`の実行
   - Sambaサービスの再起動
   - ログインページへリダイレクト

> **注意**: 構築には1〜3分かかります。処理中にブラウザを閉じないでください。

## 5. ログイン

### デフォルト認証情報（Mockモード）

| 項目 | 値 |
|------|------|
| ユーザー名 | `admin` |
| パスワード | `admin` |

### ログイン手順

1. `http://<サーバーIP>:8000` にアクセス
2. ユーザー名とパスワードを入力
3. **ログイン**ボタンをクリック
4. ダッシュボードにリダイレクト

### セッション管理

- JWTトークンは**8時間**後に期限切れ
- 期限切れ時に自動的にログインページへリダイレクト
- 右上の**ログアウト**ボタンで手動終了

## 6. ダッシュボード

ダッシュボードはActive Directory環境の全体状況を一目で確認できます。

### 構成

#### 統計カード（上部）
- **総ユーザー数**: ドメイン内の全ユーザーアカウント
- **グループ**: セキュリティグループ・配布グループ
- **コンピューター**: ドメイン参加マシン
- **組織単位**: コンテナ構造
- **GPO**: グループポリシーオブジェクト
- **ドメインコントローラー**: DC数

#### ログイントレンドチャート
- 過去7日間のログイン活動の棒グラフ

#### システムリソース
- **CPU使用率**: 現在のプロセッサ使用量
- **メモリ**: RAM使用率
- **ディスク**: ストレージ使用量

#### サービス状態
重要なADサービスのリアルタイム状態:
- LDAP（ポート389）
- Kerberos（ポート88）
- DNS（ポート53）
- SMB（ポート445）

#### OU別ユーザー分布
組織単位別のユーザー数 — 視覚的な棒グラフ。

#### 最近のアラート
最新のセキュリティイベント（アカウントロックアウト、ログイン失敗など）。

## 7. ユーザー管理

アクセス: **サイドバー → ユーザー**

### ユーザー一覧
- 1ページ50件表示
- 項目: ユーザー名、表示名、メール、部署、状態、OU、最終ログイン

### 検索とフィルタ
- **検索バー**: ユーザー名、表示名、メールでフィルタリング
- **OUフィルタ**: 特定OUのユーザーを表示
- **状態フィルタ**: 全て / アクティブ / 無効 / ロック

### 操作

#### ユーザー作成
1. **+ ユーザー追加**をクリック
2. 情報を入力:
   - **ユーザー名**（必須）: ログインID（例: `jsato`）
   - **表示名**（必須）: フルネーム（例: `佐藤次郎`）
   - **メール**: メールアドレス
   - **部署**: 部署名
   - **役職**: 役職
   - **OU**: 対象組織単位
   - **パスワード**: 初期パスワード（最小8文字）
3. **作成**をクリック

#### ユーザー編集
1. ユーザー行をクリック
2. ドロワーでフィールドを編集
3. **保存**をクリック

#### パスワードリセット
1. ユーザーを選択
2. **パスワードリセット**をクリック
3. 新しいパスワードを入力

#### 無効化 / 有効化
1. ユーザーを選択
2. **無効化**または**有効化**をクリック

#### ユーザー削除
1. ユーザーを選択
2. **削除**をクリック
3. 確認

> **警告**: 削除は元に戻せません。後で必要になる可能性がある場合は、アカウントを無効化してください。

#### CSVエクスポート
**エクスポート**をクリックして全ユーザーをCSVファイルとしてダウンロード。

## 12. DNS管理

アクセス: **サイドバー → DNS**

### ゾーン一覧
- フォワードゾーン（例: `corp.example.com`）
- リバースゾーン（例: `1.168.192.in-addr.arpa`）

### 操作

#### レコード表示
1. ゾーンを選択
2. 全DNSレコードを確認（A、CNAME、MX、SRV、TXT、PTR、SOA、NS）

#### レコード追加
1. ゾーンを選択
2. **+ レコード追加**をクリック
3. 情報を入力:
   - **名前**: レコード名（ゾーンルートは`@`を使用）
   - **タイプ**: レコードタイプ（A、CNAME、MXなど）
   - **値**: IPアドレスまたはホスト名
   - **TTL**: Time to Live（デフォルト: 3600）
4. **作成**をクリック

#### レコード削除
1. レコードを見つける
2. ゴミ箱アイコンをクリック
3. 削除を確認

## 13. ドメインポリシー

アクセス: **サイドバー → ポリシー**

### パスワードポリシー

| 設定 | 説明 | デフォルト |
|------|------|-----------|
| 複雑なパスワード | 大文字・小文字・数字・記号を要求 | 有効 |
| 最小文字数 | 最小パスワード文字数 | 7 |
| パスワード履歴 | 過去N個のパスワードを記憶 | 24 |
| 最大有効期間（日） | パスワード変更間隔 | 42 |
| 最小有効期間（日） | パスワード変更までの日数 | 1 |

### アカウントロックアウトポリシー

| 設定 | 説明 | デフォルト |
|------|------|-----------|
| ロックアウトしきい値 | ロックアウトまでの失敗回数 | 0（無効） |
| ロックアウト期間（分） | アカウントロック維持時間 | 30 |
| カウンターリセット（分） | 失敗カウンター初期化時間 | 30 |

### ポリシー更新
1. 目的のフィールドを修正
2. **保存**をクリック
3. 新しいセッションから即座に適用

## 16. APIリファレンス

完全なインタラクティブAPIドキュメントは以下で確認できます:
- **Swagger UI**: `http://<サーバーIP>:8000/docs`
- **ReDoc**: `http://<サーバーIP>:8000/redoc`
- **OpenAPI JSON**: `http://<サーバーIP>:8000/openapi.json`

### 認証

`/health`、`/auth/login`、`/setup/*`を除く全APIエンドポイントにJWT Bearerトークンが必要です:

```bash
# トークンの取得
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin"}'

# トークンの使用
curl http://localhost:8000/api/v1/users \
  -H "Authorization: Bearer <トークン>"
```

### 主なエンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| POST | `/api/v1/auth/login` | ログイン |
| GET | `/api/v1/dashboard/stats` | ダッシュボード概要 |
| GET | `/api/v1/users` | ユーザー一覧 |
| POST | `/api/v1/users` | ユーザー作成 |
| GET | `/api/v1/groups` | グループ一覧 |
| GET | `/api/v1/computers` | コンピューター一覧 |
| GET | `/api/v1/ou` | OU一覧 |
| GET | `/api/v1/gpo` | GPO一覧 |
| GET | `/api/v1/dns/zones` | DNSゾーン一覧 |
| GET | `/api/v1/logs` | システムログ |
| GET | `/api/v1/policies/domain` | ドメインポリシー |

## 17. トラブルシューティング

### サービスが起動しない場合

```bash
# サービス状態確認
sudo systemctl status samba-ad-manager

# ログ確認
sudo journalctl -u samba-ad-manager -n 50

# ポート8000の使用確認
sudo ss -tlnp | grep 8000
```

### Webポータルにアクセスできない場合

1. サービス実行確認: `systemctl is-active samba-ad-manager`
2. ファイアウォールルール確認: `sudo ufw status`
3. ネットワーク接続確認: `ping <サーバーIP>`
4. `.env`ファイルの設定確認

### ログイン失敗

- **Mockモード**: デフォルト認証情報は`admin`/`admin`
- **LDAPモード**: Samba DCの実行確認: `samba -V`

### フロントエンドが空白ページの場合

1. フロントエンドの再ビルド: `cd frontend && npm run build`
2. サービスの再起動: `sudo systemctl restart samba-ad-manager`
3. ブラウザキャッシュのクリア

### Mockモードに戻す

`.env`ファイルの編集:
```
APP_MODE=mock
```
再起動: `sudo systemctl restart samba-ad-manager`
